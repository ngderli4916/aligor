Hotmail / Outlook 邮箱接给 AI Agent

照着做就可以的小白流程

这页不是给工程师炫技术的。它是给普通人看的操作流程：你要先告诉 AI Agent 什么，复制哪一段话给它，然后跟着它去 Microsoft 开权限，最后设置成安全的只读邮箱自动化。

一句话

这个流程在做什么

你把 Hotmail / Outlook 邮箱接给 AI Agent。AI Agent 以后可以帮你看邮件标题、分类、提醒和整理。

重点：先只读。不要一开始就让它发送、删除或批量移动邮件。

谁做什么

人和 AI 的分工

人负责：登录 Microsoft、点击授权、确认账号、批准危险动作。

AI Agent 负责：教你点哪里、生成链接、测试连接、写自动化规则。

第 1 步：先准备这些资料

你先告诉 AI Agent 这些就够了。

  • 你要连接的邮箱，例如 yourname@hotmail.com。
  • 你要 AI Agent 做什么，例如“每天整理未读邮件”。
  • 你要不要只读模式。新手请选择只读。
  • 你有没有 Microsoft Azure Portal 账号。
  • 你要把 token 存在哪里，例如 1Password、macOS Keychain、环境变量或服务器 secret。

不要把邮箱密码给 AI Agent。这个流程不需要邮箱密码。

第 2 步：复制这段给你的 AI Agent

把下面整段复制给 AI Agent。它就会一步一步带你做。

请一步一步指导我把 Hotmail / Outlook 邮箱连接给 AI Agent 使用。

我的目标：
1. 先只读邮件，不发送，不删除，不批量移动。
2. AI Agent 可以读取邮件标题、发件人、时间和必要的邮件内容。
3. 读取邮件时要尽量使用 BODY.PEEK，避免把未读邮件变成已读。
4. 高风险动作必须先问我批准。

请你先问我需要准备的资料。
然后指导我去 Microsoft Azure Portal 创建 App Registration。

需要的权限包括：

第 3 步：AI Agent 通常会问你什么

如果 AI Agent 问这些，是正常的。

AI Agent 会问 你怎么回答
你要连接哪个邮箱？ 给它邮箱地址，例如 yourname@hotmail.com。
你要做只读还是允许操作？ 新手回答：先只读。
你有 Azure Portal 吗？ 有就继续。没有就先注册或叫技术人员处理。
token 要存哪里？ 回答：1Password、Keychain、环境变量或服务器 secret。
可以读取完整邮件内容吗？ 先回答：只在需要时读取，而且先读 header。

第 4 步：人要去 Microsoft 点这些

  1. 打开 Azure Portal。
    去 Microsoft Azure Portal，找到 App registrations。
  2. 创建 App。
    点 New registration，名字可以写 Email AI Agent。
  3. 复制 Client ID。
    创建后复制 Application client ID，给 AI Agent 设置成 CLIENT_ID。
  4. 去 API permissions。
    点 API permissions，再点 Add a permission。
  5. 用 Search 找权限。
    不要自己慢慢翻。用搜索框找 Office 365 Exchange Online。
  6. 加 IMAP 权限。
    选择 Delegated permissions，搜索 IMAP，勾选 IMAP.AccessAsUser.All。
  7. 加离线权限。
    搜索并加入 offline_access。这个权限让 AI Agent 以后不用每次叫你重新登录。
  8. 授权登录。
    AI Agent 会给你一个 Microsoft 登录链接。你自己打开，登录正确邮箱，然后点同意。
  9. 把授权结果交回 AI Agent。
    AI Agent 会用授权结果换 refresh_token，然后安全保存。

第 5 步：连接成功后怎么 Setting

连接成功后，让 AI Agent 帮你设置这些。

  • CLIENT_ID：Microsoft App 的 client ID。
  • EMAIL：你的 Hotmail / Outlook 邮箱。
  • REFRESH_TOKEN：授权后拿到的长期 token，要安全保存。
  • IMAP_HOST：固定写 outlook.office365.com。
  • IMAP_PORT：固定写 993。
  • SCOPE：https://outlook.office.com/IMAP.AccessAsUser.All offline_access。
  • READONLY：先设成 true。

最安全的第一版设置：只读收件箱，每天检查一次，只汇报重要邮件，不自动删除，不自动发送。

第 6 步：先做一个安全测试

叫 AI Agent 先测试这些，不要直接做完整自动化。

  1. 测试 token。
    确认 refresh_token 可以换到新的 access_token。
  2. 测试 IMAP 登录。
    连接 outlook.office365.com:993。
  3. 只读打开 INBOX。
    使用 readonly 模式，不改变邮箱状态。
  4. 只读 5 封邮件 header。
    读取发件人、标题、日期和 Message-ID。
  5. 确认未读状态没有变。
    如果未读邮件变成已读，马上停下来修正读取方式。

一定要告诉 AI Agent 的安全规则

  • 默认只读。
  • 优先读取邮件 header。
  • 需要正文时使用 BODY.PEEK。
  • 不要自动发送邮件。
  • 不要自动删除邮件。
  • 不要自动退订邮件。
  • 不要读取银行、验证码、Microsoft 安全通知后转发给外部服务。
  • 任何高风险动作，都要先问人。

AI Agent 最后会用到的技术设置

这一段不是给小白手打的，是给 AI Agent 或技术人员检查用。

EMAIL=yourname@hotmail.com
CLIENT_ID=从 Azure App Registration 复制
REFRESH_TOKEN=从安全存储读取
TOKEN_URL=https://login.microsoftonline.com/common/oauth2/v2.0/token
SCOPE=https://outlook.office.com/IMAP.AccessAsUser.All offline_access
IMAP_HOST=outlook.office365.com
IMAP_PORT=993
READONLY=true

读取邮件时优先使用：
BODY.PEEK[HEADER.FIELDS (FROM SUBJECT DATE MESSAGE-ID)]

需要正文时使用：
BODY.PEEK[TEXT]

完成后你应该得到什么

  • AI Agent 可以登录 IMAP。
  • AI Agent 可以读取邮件标题和发件人。
  • 未读邮件不会被自动变成已读。
  • refresh token 已经安全保存。
  • 自动化规则已经写清楚。
  • 发送、删除、退订这些动作还是需要人批准。

记住一句话：AI Agent 负责帮你做重复工作，人负责授权和危险决定。
