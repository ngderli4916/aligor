AI agent email setup

How to link an AI agent with a personal Hotmail account

Use Microsoft OAuth2 so the account owner signs in safely, gives consent, and your AI agent can connect to Hotmail through IMAP without collecting the email password.

OAuth2 only Personal Hotmail / Outlook.com IMAP access
Standard Structure

Standard Structure

This guide is for a personal Microsoft account such as Hotmail, Outlook.com, or Live.com. The clean setup is: register an app, request delegated mail scopes, sign the user in with Microsoft, exchange the authorization code for tokens, then connect to IMAP with XOAUTH2.

Setup Steps

OAuth2 + IMAP
01

Register your Microsoft app

Create an app in Microsoft Entra. Choose personal Microsoft accounts, or both work and personal accounts if your agent supports both. Add your real redirect URL, such as your agent callback page.

02

Add delegated permissions

Request only what you need. For IMAP reading, use offline_access and IMAP.AccessAsUser.All. Add SMTP.Send only if your agent must send email.

03

Send the user to Microsoft login

Open the Microsoft authorization URL with your client_id, redirect_uri, response_type=code, scopes, and state. For public clients, use PKCE.

04

Exchange the code for tokens

Your backend exchanges the authorization code at the token endpoint. Store refresh tokens securely and use them to get fresh access tokens when needed.

05

Connect to Outlook IMAP

Connect to outlook.office365.com on port 993 with SSL/TLS. Authenticate with XOAUTH2 using the user email and the access token.

06

Refresh and protect access

Access tokens expire. Refresh them quietly, encrypt stored tokens, and give the user a way to disconnect the agent.

Important Values

Authorization endpoint
https://login.microsoftonline.com/consumers/oauth2/v2.0/authorize
Token endpoint
https://login.microsoftonline.com/consumers/oauth2/v2.0/token
IMAP host
outlook.office365.com
IMAP port
993 SSL/TLS
Main scopes
offline_access https://outlook.office.com/IMAP.AccessAsUser.All
XOAUTH2 format
user=<email>\x01auth=Bearer <access_token>\x01\x01

Safety Rules

  • Never ask users to type their Hotmail password into your AI agent.
  • Use the least permissions needed for the agent task.
  • Encrypt refresh tokens and do not expose them to the browser.
  • Let users revoke or disconnect access anytime.

Quick Troubleshooting

  • AUTHENTICATE failed usually means the token is expired, the scope is wrong, or the client is not using XOAUTH2.
  • invalid_grant usually means the code expired, PKCE failed, or the redirect URI does not match.
  • For personal Hotmail accounts, use the consumers Microsoft login tenant.
