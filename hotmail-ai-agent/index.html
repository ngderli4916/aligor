Working setup from my package

Hotmail / Outlook email automation with IMAP + OAuth2 + Python

This version follows the successful method from my package: register an Azure app, get a refresh token through Microsoft login, save it safely, then let Python refresh access tokens and connect to IMAP with XOAUTH2.

Refresh token method Hotmail / Outlook.com IMAP XOAUTH2
The Working Flow

The Working Flow

The successful flow is: Refresh Token -> Microsoft token endpoint -> Access Token -> IMAP XOAUTH2 login -> read or organize mailbox. The script should save only the refresh token securely, refresh the access token automatically, and read headers with BODY.PEEK so emails are not marked as read.

Setup Steps

Azure + Token + Python
01

Register the app in Azure Portal

Open Azure Portal -> App registrations -> New registration. Name it something like Hotmail IMAP Client. For account type, choose personal Microsoft accounts only, or any Microsoft ID if you want broader support.

02

Use the native client redirect URI

Set Redirect URI type to Public client/native and use https://login.microsoftonline.com/common/oauth2/nativeclient. This matches the package method for manually collecting the authorization code.

03

Copy the client ID and do not create a secret

After registration, copy Application client ID. Because this is a public/native client flow, you do not need a client secret. Do not put secrets or real tokens inside the website or GitHub repo.

04

Add IMAP delegated permission

In API permissions, add delegated permission IMAP.AccessAsUser.All and make sure offline_access is included. These are needed so the script can read mail through IMAP and refresh access later.

05

Get the authorization code in the browser

Open the authorize URL with your client ID, nativeclient redirect URI, response_type=code, and the IMAP/offline_access scopes. After Microsoft login, copy the code value from the redirected URL.

06

Use cURL to exchange code for refresh token

POST the code to Microsoft token endpoint with grant_type=authorization_code. Save the refresh_token from the JSON response into macOS Keychain, then Python can refresh access tokens whenever it connects.

Very Important Rules

  • Mask real emails, client IDs, access tokens, and refresh tokens before sharing screenshots or code.
  • Store refresh tokens in macOS Keychain or a real secret manager, not inside scripts or GitHub.
  • Use BODY.PEEK for reading headers so emails are not accidentally marked as read.
  • Do not run permanent delete or automatic mailto unsubscribe without human approval.

Get Refresh Token

First open the authorize URL in a browser. After login, copy the code from the address bar. Then exchange it with cURL.

https://login.microsoftonline.com/common/oauth2/v2.0/authorize?
client_id=YOUR_CLIENT_ID&
response_type=code&
redirect_uri=https://login.microsoftonline.com/common/oauth2/nativeclient&
scope=https://outlook.office.com/IMAP.AccessAsUser.All offline_access
curl -X POST https://login.microsoftonline.com/common/oauth2/v2.0/token \
  -d "client_id=YOUR_CLIENT_ID" \
  -d "grant_type=authorization_code" \
  -d "code=CODE_FROM_BROWSER" \
  -d "redirect_uri=https://login.microsoftonline.com/common/oauth2/nativeclient" \
  -d "scope=https://outlook.office.com/IMAP.AccessAsUser.All offline_access"

Save Token in macOS Keychain

The package stores the refresh token in Keychain, then Python reads it when refreshing the access token.

security add-generic-password -U \
  -s "himalaya-cli" \
  -a "hotmail-imap-refresh-token" \
  -w "YOUR_REFRESH_TOKEN"
security find-generic-password \
  -s "himalaya-cli" \
  -a "hotmail-imap-refresh-token" \
  -w

Python Connection Logic

Python reads the refresh token, calls Microsoft token endpoint, receives a fresh access token, then builds the XOAUTH2 string for IMAP login.

ACCOUNT = "your_email@hotmail.com"
CLIENT_ID = "741e...fbba"
TOKEN_URL = "https://login.microsoftonline.com/common/oauth2/v2.0/token"
IMAP_HOST = "outlook.office365.com"
IMAP_PORT = 993

auth = f"user={ACCOUNT}\x01auth=Bearer {access_token}\x01\x01"
imap = imaplib.IMAP4_SSL(IMAP_HOST, IMAP_PORT)
imap.authenticate("XOAUTH2", lambda _: auth.encode())

Read Headers Only

The working package reads only selected headers and uses BODY.PEEK. This keeps the mailbox private, faster, and avoids marking messages as read.

imap.uid(
  "FETCH",
  chunk,
  "(BODY.PEEK[HEADER.FIELDS (FROM SUBJECT DATE MESSAGE-ID)])",
)

If It Fails, Check These First

  • XOAUTH2 authentication failed: access token expired, wrong scope, or refresh token was not updated.
  • AADSTS50158: password/ROPC flow is blocked by MFA or Microsoft security. Use the browser authorization code method.
  • AADSTS700016: app registration or client ID is wrong. Check the Azure Application client ID.
  • Emails became read: the code used BODY[] instead of BODY.PEEK[]. Replace it globally.
