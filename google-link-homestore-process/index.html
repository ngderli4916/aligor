Google Link 连接 Website 与 HomeStore 操作记录

这份记录整理今天把 Google Authentication / Google Sheet / Google Apps Script / Cloudflare Worker 接到 website 与 HomeStore 相关流程的做法。用途：以后放到 website，当成项目记录、教学内容或 Codex 继续加工的素材。

记录时间：2026-06-04 18:15 +0800 项目：Aligor / 阿理哥 主题：Website → Google Sheet Lead Capture 原则：不公开 secrets / token / shared secret

1. 最终做到的结果

今天完成的重点不是单纯把一个 link 放到网页，而是让一个完整的数据流程可以跑起来：

Website Form：访客在网页填写报名资料。
Cloudflare Worker：网页把 form data 送到 Worker，Worker 做中间层。
Google Apps Script Web App：Worker 再把资料转送到 Google Apps Script。
Google Sheet：Apps Script 把资料 append 到指定 Google Sheet。
Hermes / CRM：之后 AI agent 可以读取 Sheet，整理新 lead 给 Adrian 审核。
WhatsApp / Email Follow-up：Adrian 批准后，才做 WhatsApp 邀请与 Email 确认。

验证结果：有人提交 form 后，Google Sheet 已经成功增加报名名单，代表 Website → Cloudflare → Google → Sheet 的链路已经通。

2. 这套架构为什么这样接

Website

负责展示内容和收集用户输入。网页不直接暴露 Google credential，也不直接把 sensitive endpoint 写死成公开秘密。

Cloudflare Worker

作为安全中间层。它接收 website 的 form submission，再转发去 Google Apps Script。这样以后要换 backend、加验证、加通知，都可以在 Worker 处理。

Google Apps Script

作为 Google 世界里的 bridge。它有权限写入 Google Sheet，也可以之后扩展成发内部通知 email。

Google Sheet

作为 early-stage CRM。好处是透明、容易检查、容易修改 status，适合刚开始验证 webinar funnel。

3. Google Authentication 的准备

为了让本机 AI agent / Hermes 可以读取 Google Sheet、Drive、Calendar、Gmail 等资料，先完成 Google OAuth authentication。

  1. 在 Google Cloud Console 建立或选择一个 project。
  2. 开启需要的 Google API，例如：Google Sheets API、Google Drive API、Google Calendar API、Gmail API、Google Docs API。
  3. 设定 Google Auth Platform / OAuth consent screen。
  4. 建立 OAuth Client，类型使用 Desktop app。
  5. 下载 OAuth client JSON。
  6. 在本机用 Google Workspace setup script 跑 OAuth 授权。
  7. 打开授权 URL，用自己的 Google account approve。
  8. 把 browser redirect 回来的 code / URL 交给 setup script 换 token。
  9. 最后检查 token 是否有效。
验证方式：
setup.py --check

成功状态：
AUTHENTICATED

安全记录：OAuth client secret、refresh token、access token、Apps Script shared secret 都不能写进 website，也不要放进公开 repo。这里只记录流程，不记录秘密值。

5. Google Apps Script 做了什么

Google Apps Script 是接收资料和写入 Sheet 的地方。核心逻辑：

  1. 接收 POST request。
  2. 读取 form data，例如 name、whatsapp、email、city、role、pain point。
  3. 检查 shared secret（如果有设）。
  4. 打开指定 Google Sheet。
  5. 确保 Sheet header 存在。
  6. 把报名资料 append 成新 row。
  7. 可选：发送内部通知 email 给管理员。
  8. 回传 JSON：{ ok: true }。
简化版逻辑：
doPost(e)
→ params = e.parameter
→ verify form_secret
→ open spreadsheet
→ get / create sheet "Webinar Leads"
→ append row
→ return { ok: true }

Apps Script Web App URL 应该当成 backend endpoint 使用。不要把 shared secret 写在公开页面里。

6. Cloudflare Worker 做了什么

Cloudflare Worker 是 website 和 Google Apps Script 中间的保护层。

  1. Website submit form 到 Worker endpoint。
  2. Worker 处理 CORS，限制 allowed origins。
  3. Worker 检查 request method，允许 POST / OPTIONS。
  4. Worker 从 environment variable 读取 GOOGLE_APPS_SCRIPT_URL。
  5. 如果有 FORM_SHARED_SECRET，Worker 自动加进 form data。
  6. Worker 用 fetch() 把 form data 转发给 Google Apps Script。
  7. Google 成功后，Worker 回传 { ok: true } 给网站。
Website
  POST formData
    ↓
Cloudflare Worker
  add secret from env
  forward to Google Apps Script
    ↓
Google Sheet append row

7. Website 连接 Google Link 的方式

Website 不是直接连接 Google Sheet，而是连接 Cloudflare Worker endpoint。这样比较安全，也比较容易维护。

网页 form submit flow

  1. 用户填写报名表。
  2. JavaScript 收集 form data。
  3. 补上 tracking data，例如 landing page URL、referrer、utm、fbclid、source page。
  4. 送到 Cloudflare Worker endpoint。
  5. 收到成功 response 后，在页面显示报名成功讯息。
前端概念：
const formData = new FormData(form)
formData.set('submitted_at', new Date().toISOString())
formData.set('landing_page_url', window.location.href)
formData.set('referrer', document.referrer || '')

fetch(WORKER_ENDPOINT, {
  method: 'POST',
  body: formData
})

重点：Website success message 只代表前端收到成功 response；真正验证要去 Google Sheet 看有没有新增 row。

8. HomeStore 连接 Google Link 的方式

HomeStore 的操作可以沿用同一套 Google Link 思路：不要让 HomeStore 直接暴露 Google token，而是让 HomeStore 把资料送到中间层，再由中间层写入 Google Sheet 或 Google 服务。

HomeStore 推荐连接方式

  1. HomeStore 页面 / 系统产生用户资料或订单资料。
  2. HomeStore submit 到 Cloudflare Worker endpoint。
  3. Worker 判断来源、加入 secret、整理字段。
  4. Worker 转发到 Google Apps Script。
  5. Apps Script 写入指定 Google Sheet tab。
  6. Hermes / AI agent 再从 Sheet 读取资料，做后续整理、提醒或 follow-up。
HomeStore
→ Cloudflare Worker
→ Google Apps Script
→ Google Sheet
→ Hermes / CRM follow-up

记录说明：如果 HomeStore 使用的是另一个 page、另一个 Sheet 或另一个 Apps Script endpoint，正式放上 website 前，把名称和 URL 替换成实际版本即可；不要把 secret 放进去。

9. CRM / Follow-up 规则

Google Sheet 接到 lead 后，不代表立刻自动 WhatsApp 或 Email。现阶段使用人工审核比较安全。

Step 1：Hermes 检查 Google Sheet 是否有新 lead。
Step 2：Hermes 在 Telegram 汇报给 Adrian 审核。
Step 3：Adrian 确认是否发送 WhatsApp。
Step 4：发送前确认使用哪一个 WhatsApp connection / from number。
Step 5：Adrian 批准后才 send WhatsApp 邀 group。
Step 6：对方 join / OK 后，再发正式 Email 确认。

重要：不要自动 send WhatsApp。必须先问 Adrian，因为 sender / WhatsApp connection 可能会更换。

10. 验证 Checklist

  • ✓ Google OAuth authentication 成功。
  • ✓ Google API 可以读取 Sheet / Drive / Calendar 等服务。
  • ✓ Website form 可以 submit。
  • ✓ Cloudflare Worker 可以接收 POST。
  • ✓ Worker 可以转发到 Google Apps Script。
  • ✓ Apps Script 可以 append row 到 Google Sheet。
  • ✓ Google Sheet 已经出现真实报名名单。
